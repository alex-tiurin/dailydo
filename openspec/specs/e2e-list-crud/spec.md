## ADDED Requirements

### Requirement: Создание нового дня (списка)
Пользователь SHALL иметь возможность создать новый список задач через экран Create New List.

#### Scenario: Список появляется на главном экране после создания
- **WHEN** пользователь заполняет имя списка на `/create` и нажимает `save-list-button`
- **THEN** приложение SHALL перейти на `/` и новая карточка SHALL быть видна в списке дней

#### Scenario: Нельзя создать список без имени
- **WHEN** пользователь нажимает `save-list-button` с пустым полем имени
- **THEN** приложение SHALL остаться на `/create` и отобразить сообщение об ошибке валидации

### Requirement: Удаление дня (списка)
Пользователь SHALL иметь возможность удалить существующий список.

#### Scenario: Список исчезает после удаления
- **WHEN** пользователь выполняет действие удаления для конкретного списка
- **THEN** карточка этого дня SHALL исчезнуть с главного экрана `/`

#### Scenario: При удалении последнего списка отображается empty state
- **WHEN** пользователь удаляет единственный оставшийся список
- **THEN** SHALL отобразиться `empty-state` с кнопкой `create-first-list-button`

### Requirement: Исследовательское тестирование через Playwright CLI
Перед написанием e2e-тестов команда SHALL провести исследовательское тестирование с помощью `playwright codegen` для:

#### Scenario: Исследование UI на предмет реализованных фич
- **WHEN** запускается `playwright codegen http://localhost:3000`
- **THEN** тестировщик SHALL задокументировать: наличие/отсутствие кнопки удаления списка, поведение кнопки `task-edit-*`, наличие скрытых UI-элементов (hover, context menu)

#### Scenario: Генерация базовых тестов через codegen
- **WHEN** тестировщик проходит user flow в Playwright Inspector
- **THEN** SHALL быть сгенерированы базовые тесты, которые затем рефакторятся под PO + Step pattern
