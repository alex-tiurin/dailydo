## MODIFIED Requirements

### Requirement: Переключение статуса задачи (toggle done)
Пользователь SHALL иметь возможность отмечать задачи как выполненные и возвращать их в pending через стабильные селекторы без runtime-id. Тестовые действия SHALL находить задачу по имени и переключать `task-checkbox` внутри соответствующей строки.

#### Scenario: Pending задача перемещается в Completed
- **WHEN** пользователь кликает `task-checkbox` у задачи в секции Pending, выбранной по имени задачи
- **THEN** задача SHALL переместиться в секцию `completed-section` и отображаться с зачёркиванием

#### Scenario: Completed задача возвращается в Pending
- **WHEN** пользователь кликает `task-checkbox` у задачи в секции Completed, выбранной по имени задачи
- **THEN** задача SHALL переместиться в секцию `pending-section` и отображаться без зачёркивания

#### Scenario: Счётчик прогресса обновляется после toggle
- **WHEN** пользователь переключает статус задачи
- **THEN** `progress-counter` SHALL отобразить обновлённое соотношение done/total

### Requirement: Редактирование задачи (переименование)
Пользователь SHALL иметь возможность изменить название задачи через кнопку редактирования со стабильным селектором без runtime-id.

#### Scenario: Открытие режима редактирования задачи
- **WHEN** пользователь кликает `task-edit` у строки задачи, выбранной по имени
- **THEN** SHALL появиться поле ввода или модал для изменения названия задачи

#### Scenario: Сохранение нового названия задачи
- **WHEN** пользователь вводит новое название и подтверждает изменение
- **THEN** `task-name` у изменённой строки SHALL отобразить обновлённое название
