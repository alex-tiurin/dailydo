## ADDED Requirements

### Requirement: E2E тест SHALL верифицировать флоу создания первого списка из пустого состояния
Тест `empty_state_create_list` SHALL запускаться с чистого состояния, верифицировать empty state и создать список "Morning Routine" с двумя задачами, затем убедиться, что карточка появилась на My Lists.

#### Scenario: Создание первого списка из empty state
- **WHEN** приложение запускается без данных (после restart)
- **THEN** экран My Lists показывает empty state с кнопкой "Create First List"
- **WHEN** пользователь нажимает "Create First List", вводит имя "Morning Routine", добавляет задачи "Exercise" и "Breakfast", нажимает "Save List"
- **THEN** экран My Lists показывает карточку "Morning Routine" с 2 задачами

### Requirement: E2E тест SHALL верифицировать переключение статуса задачи в Progress View
Тест `task_toggle_completion` SHALL открыть Progress View существующего списка, переключить чекбокс первой задачи и убедиться, что задача переместилась в секцию Completed.

#### Scenario: Переключение задачи в выполненную
- **WHEN** на My Lists есть список с задачами и пользователь нажимает на карточку
- **THEN** открывается Progress View со всеми задачами в секции Pending
- **WHEN** пользователь нажимает чекбокс первой задачи
- **THEN** задача появляется в секции Completed, счётчик выполненных обновляется

### Requirement: E2E тест SHALL верифицировать навигацию между всеми экранами
Тест `navigation_flow` SHALL проверить полный навигационный граф: My Lists → Create New List → back, My Lists → Progress View → back.

#### Scenario: Навигация между всеми экранами приложения
- **WHEN** пользователь нажимает кнопку создания нового списка на My Lists
- **THEN** открывается экран Create New List
- **WHEN** пользователь нажимает "Back"
- **THEN** отображается экран My Lists
- **WHEN** пользователь нажимает на карточку списка
- **THEN** открывается Progress View
- **WHEN** пользователь нажимает кнопку "Back"
- **THEN** отображается экран My Lists

### Requirement: E2E тест SHALL верифицировать создание списка с несколькими задачами
Тест `create_list_multiple_tasks` SHALL создать список "Work Tasks" с тремя задачами и проверить корректное отображение счётчика задач на карточке My Lists.

#### Scenario: Создание списка с 3 задачами и верификация счётчика
- **WHEN** пользователь создаёт список "Work Tasks" с задачами "Email", "Meeting", "Report"
- **THEN** на My Lists карточка "Work Tasks" показывает 3 задачи
- **WHEN** пользователь открывает список
- **THEN** Progress View показывает 3 задачи в секции Pending и счётчик "0/3"

### Requirement: E2E тест SHALL верифицировать полный рабочий сценарий дня
Тест `full_day_workflow` SHALL выполнить создание списка, последовательное выполнение всех задач и верификацию 100% прогресса.

#### Scenario: Полный рабочий день — создание и выполнение всех задач
- **WHEN** пользователь создаёт список "Daily Goals" с задачами "Reading" и "Exercise"
- **THEN** карточка появляется на My Lists
- **WHEN** пользователь открывает список и переключает оба чекбокса
- **THEN** обе задачи в секции Completed, счётчик показывает "2/2" (100%)
