# Findings: Нереализованные фичи и потенциальные баги

Документ составлен по результатам исследовательского тестирования webapp (2026-04-04).

---

## Нереализованные фичи

### 1. Кнопка "Back" / "Cancel" на экране Create New List

**Экран:** `/create`  
**Проблема:** Нет способа вернуться на главный экран без сохранения списка. Пользователь вынужден использовать браузерную кнопку Back.  
**Есть в UI:** нет  
**Связанный тест:** `tests/e2e/navigation.spec.ts` → `test.skip: navigates to /create from empty state button`

---

### 2. Удаление списка (дня)

**Экран:** `/` (My Lists)  
**Проблема:** API и context полностью реализованы, но в UI нет кнопки или жеста для удаления.  
Иконка карандаша на карточке дня — декоративный `<img>` без обработчика.  
**Есть в API:** `DELETE /api/lists/:id` ✅  
**Есть в context:** `deleteList(id)` ✅  
**Есть в UI:** ❌  
**Связанные тесты:** `tests/e2e/list-crud.spec.ts` → 2 × `test.skip`

---

### 3. Редактирование названия задачи

**Экран:** `/list/:id` (Progress View)  
**Проблема:** Кнопка-карандаш `task-edit-{id}` присутствует в DOM для каждой задачи, но не имеет `onClick` обработчика. Нет ни модала, ни инлайн-редактирования.  
**Есть в API:** `PATCH /api/lists/:id/tasks/:taskId` с полем `name` ✅  
**Есть в UI:** кнопка есть, обработчик отсутствует ❌  
**Связанный тест:** `tests/e2e/task-crud.spec.ts` → `test.skip: edits a task name via inline edit`

## Потенциальные баги

### 4. Отсутствие секций при крайних состояниях Progress View

**Экран:** `/list/:id`  
**Проблема:** Секции `pending-section` и `completed-section` рендерятся условно — только если есть задачи соответствующего статуса. Если все задачи выполнены — `pending-section` исчезает из DOM. Если ни одна не выполнена — исчезает `completed-section`.  
**Риск:** Тесты и сторонний код, ожидающие оба элемента всегда в DOM, упадут.  
**Воспроизведение:** открыть список, где все задачи в одном статусе.

---

### 5. Нет кнопки сброса / очистки данных в тестовой среде

**Тип:** ограничение тестируемости  
**Проблема:** In-memory backend не имеет эндпоинта сброса (`POST /api/reset` или аналога). Состояние накапливается между тестовыми сессиями — e2e-тесты не могут проверить empty state без перезапуска сервера.  
**Связанный тест:** `tests/e2e/navigation.spec.ts` → `test.skip: navigates to /create from empty state button`
