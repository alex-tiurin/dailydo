# User Stories & Test Cases

Источник: `webapp/tests/e2e/` (Playwright, прогон против реального приложения с in-memory бэкендом, без мокирования API).

Легенда статусов:
- ✅ — активный тест, реализован и проходит
- ⏭️ — `test.skip`: функциональность в UI не реализована, тест подготовлен

---

## 1. CRUD списков (дней)

Файл: `webapp/tests/e2e/list-crud.spec.ts`

### TC-L-01 ✅ Создание нового списка и появление карточки на My Lists
**As a** пользователь
**I want to** создать новый список задач
**So that** он появится в виде карточки на главном экране.

**Шаги:**
1. Открыть страницу My Lists.
2. Нажать кнопку `+ New List` и перейти на `/create`.
3. Ввести имя списка и текст первой задачи.
4. Сохранить список.
5. Проверить что список отображается на главном экране.
5. Открыть созданный список и проверить заголовок и элементы списка

**Ожидаемый результат:** карточка отображается на главном экране и ее данные корректно отображаются в Progress View

---

### TC-L-02 ✅ Валидация пустого имени списка
**As a** пользователь
**I want to** получить ошибку валидации при попытке сохранить список без имени
**So that** не создавать пустые записи.

**Шаги:**
1. Открыть My Lists → перейти на Create New List.
2. Нажать Save без ввода имени списка.

**Ожидаемый результат:** пользователь остаётся на `/create`, отображается сообщение «List name is required».

---

### TC-L-03 ⏭️ Удаление списка и исчезновение карточки
**Причина пропуска:** DELETE `/api/lists/:id` реализован на бэкенде, но в UI нет кнопки удаления — иконка карандаша на DayCard декоративная, без обработчика. Тест активируется после добавления UI-триггера.

**Ожидаемый результат (при наличии UI):** карточка удалённого списка исчезает с My Lists.

---

### TC-L-04 ⏭️ Пустой state после удаления последнего списка
**Причина пропуска:** зависит от реализации удаления (см. TC-L-03).

**Ожидаемый результат (при наличии UI):** после удаления последнего списка отображается empty state с кнопкой «Create First List».

---

## 2. Навигация между экранами

Файл: `webapp/tests/e2e/navigation.spec.ts`

### TC-N-01 ✅ Переход на `/create` по кнопке `+ New List`
**As a** пользователь
**I want to** перейти к созданию нового списка с главного экрана
**So that** начать заполнение формы.

**Шаги:**
1. Открыть My Lists.
2. Нажать `+ New List`.

**Ожидаемый результат:** открывается страница Create List.

---

### TC-N-02 ✅ Переход к Progress View по клику на карточку дня
**Шаги:**
1. Создать список (предусловие данных); после save вернуться на My Lists.
2. Кликнуть на созданную карточку дня на My Lists.

**Ожидаемый результат:** открывается URL `/list?id=<id>` с Progress View, заголовком и кнопкой Back.

---

### TC-N-03 ✅ Возврат на My Lists из Progress View
**Шаги:**
1. Создать список — save автоматически открывает Progress View (`/list?id=<id>`).
2. Нажать кнопку Back.

**Ожидаемый результат:** открывается страница My Lists без крэша.

---

### TC-N-04 ✅ Полный навигационный цикл
**Сценарий:** My Lists → Create → Progress View → My Lists → Progress View → My Lists.

**Шаги:**
1. Открыть My Lists.
2. Перейти к Create, заполнить имя и задачу, сохранить — открывается Progress View.
3. Нажать Back, убедиться что новая карточка видна на My Lists.
4. Открыть её Progress View повторно.
5. Убедиться, что секция Pending видна.
6. Нажать Back и вернуться на My Lists.

**Ожидаемый результат:** каждый переход успешен, состояние сохраняется.

---

### TC-N-05 ⏭️ Переход на `/create` из empty state
**Причина пропуска:** in-memory backend сидит данными на старте и не имеет endpoint для сброса — в e2e невозможно гарантировать empty state.

**Ожидаемый результат (при наличии reset-API):** кнопка «Create First List» открывает `/create`.

## 3. CRUD задач

Файл: `webapp/tests/e2e/task-crud.spec.ts`

### TC-T-01 ✅ Создание списка с несколькими задачами и отображение их в Pending
**Шаги:**
1. Открыть My Lists → Create New List.
2. Ввести имя списка.
3. Добавить две задачи («First task», «Second task»).
4. Сохранить и дождаться редиректа на My Lists.
5. Открыть Progress View созданного списка.

**Ожидаемый результат:** в секции Pending отображаются обе задачи.

---

### TC-T-02 ✅ Перевод задачи из Pending в Completed
**Шаги:**
1. Создать список с одной задачей «Toggle me».
2. Открыть Progress View.
3. Убедиться, что задача в Pending.
4. Кликнуть по чекбоксу задачи.

**Ожидаемый результат:**
- задача отображается в секции Completed,
- секция Pending скрыта,
- счётчик прогресса обновляется до `1 / 1`.

---

### TC-T-03 ✅ Возврат задачи из Completed обратно в Pending
**Шаги:**
1. Создать список с задачей «Toggle back», открыть Progress View.
2. Отметить задачу выполненной — секция Pending пропадает, задача в Completed.
3. Повторно кликнуть по чекбоксу.

**Ожидаемый результат:**
- задача снова в Pending,
- секция Completed скрыта,
- счётчик `0 / 1`.

---

### TC-T-04 ⏭️ Inline-переименование задачи
**Причина пропуска:** кнопка `task-edit-*` присутствует в DOM, но не имеет обработчика `onClick` — rename в UI не реализован. Тест активируется после реализации фичи.

**Ожидаемый результат (при наличии фичи):** после редактирования имя задачи в Progress View обновляется, старое имя не отображается.

## Известные ограничения продукта (на 2026-04-04)

- Удаление списка: API готов, UI-триггер отсутствует (иконка карандаша на DayCard — декоративная).
- Редактирование имени задачи: кнопка есть, обработчик не привязан.
- Сброс данных для e2e: in-memory backend не предоставляет reset-endpoint, из-за чего empty state недостижим в e2e без перезапуска сервера.
